## One config

A site runs from a `nimpress.config` file with sensible defaults. The `nimpress` command serves, builds, previews, and scaffolds with `init`, so a consumer needs little beyond that file and its markdown.

## Assets and styling

Shared assets live in a root folder served at a configurable base, while images next to a page travel with it. A site wide stylesheet named in `css` and a per page stylesheet sharing a markdown file name override the theme.

## Content linting

`nimpress lint` validates the frontmatter of every page, and a build fails on an invalid page, so a broken page cannot ship.

## Docs across repos

A repo keeps its docs in a `.nimpress` folder. On change it notifies the docs site, which mirrors the folder into its tree under a mapped path, lints, and publishes.
