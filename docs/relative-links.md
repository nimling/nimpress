Authoring links between markdown pages.

## Rule

Use the resolved route, not the file path. Route resolution is the `path` field in frontmatter if present, otherwise the file location relative to the content root with `.md` stripped and `/index` collapsed.

## Within a section

A page at `docs/guide/intro.md` referencing `docs/guide/setup.md`:

```md
[Setup](./setup)
```

The runtime router resolves `./setup` relative to the current route, not the file location, so the link works whether or not the author overrode either page's `path`.

## To another section

```md
[OpenAPI renderer](/docs/openapi)
```

Absolute paths are always relative to the site root.

## Anchors

```md
[Hash deep links](./openapi#hash-deep-links)
```

Anchors point at the slug generated by `markdown-it-anchor` from the heading text.

## Cross page anchors with OpenAPI

```md
[Bookable POST endpoint](/api/bookable#operation/PostBookable)
[User schema](/api/bookable#schema/User)
[Bookings tag](/api/bookable#tag/Bookings)
```

The OpenAPI renderer wires `#operation/<id>`, `#schema/<name>`, and `#tag/<name>` to the corresponding elements.

## External links

Plain URLs. The link renders unchanged.

```md
[GitHub](https://github.com/nimling/nimpress)
```
